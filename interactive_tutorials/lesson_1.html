Variables & Types

Learn how to store and work with different types of data in Fuzion. This is the foundation of all programming!

What Are Variables?

Think of a variable as a labeled box where you can store information. Just like you might label a box "Books" or "Toys", in programming we give variables names that describe what they hold.

üéØ Key Point

A variable has two parts: a name (what you call it) and a value (what it contains).

Your First Variable

Creating a variable in Fuzion is super simple:

Example: Creating a variable 
# Create a variable called 'name' and store your name in it
name = "Alice"

# Print the value
print name
Output:

That's it! Notice a few things:

  • No special keywords needed (unlike other languages)
  • No semicolons at the end of lines
  • Comments start with #

Naming Variables

You can name variables almost anything, but follow these rules:

Good variable names 
# ‚úì Good names - clear and descriptive
user_name = "Bob"
age = 25
total_price = 99.99
is_student = true

# ‚úó Avoid these - unclear or confusing
# x = "Bob"          # What is x?
# a123 = 25          # What does a123 mean?
# TotalPrice = 99.99 # Don't use capital letters
üí° Best Practice

Use names that describe what the variable contains. Someone reading your code (including future you!) should understand what each variable is for.

Types of Data

Variables can store different types of data. Here are the main ones:

1. Text (Strings)

Text is wrapped in quotes:

Working with text 
first_name = "Alice"
last_name = "Smith"
city = "Lagos"

print first_name
print last_name
print city
Output:

2. Numbers

Numbers don't need quotes:

Working with numbers 
age = 25
price = 19.99
year = 2025

print age
print price
print year
Output:

3. True/False (Booleans)

For yes/no, on/off, true/false values:

Working with booleans 
is_student = true
has_license = false
is_raining = true

print is_student
print has_license
print is_raining
Output:

Putting Variables in Text

You can insert variables into text using {variable_name}:

String interpolation 
name = "Alice"
age = 25
city = "Lagos"

print "My name is {name}"
print "I am {age} years old"
print "I live in {city}"
print "Hi, I'm {name}, I'm {age} and I live in {city}"
Output:
üí° Pro Tip

This makes your output much more readable than joining strings with + signs!

Variables Are Immutable by Default

One unique feature of Fuzion: variables can't be changed unless you explicitly say so. This prevents accidental bugs!

This won't work (immutable by default) 
score = 100
# score = 200  # ERROR! Can't change 'score'

Making Variables Changeable

Use the change keyword if you need to modify a variable:

Mutable variables 
# Use 'change' to make it mutable
change score = 0

print "Starting score: {score}"

score = 10
print "After round 1: {score}"

score = 25
print "After round 2: {score}"

score = 50
print "Final score: {score}"
Output:
üéØ Why This Matters

Immutability by default helps prevent bugs. If you accidentally try to change a variable you didn't mean to change, Fuzion will catch the error immediately!

üí° When to Use 'change'

Only use change when you know you'll need to update the variable later. For most variables, stick with the default (immutable).

Type Inference

Fuzion is smart! It automatically figures out what type of data a variable contains:

Fuzion figures out types automatically 
name = "Alice"        # Fuzion knows this is text
age = 25               # Fuzion knows this is a number
is_student = true      # Fuzion knows this is true/false

# You don't need to tell Fuzion the type!
üí° Compare to Other Languages

In many languages, you'd write: String name = "Alice" or int age = 25. Fuzion makes it simpler!

üéØ What is a "Block Header"?

A block header is the line that introduces a new indented code block. It tells FUZION "the code below belongs to this structure."

üìò The Colon Rule (Universal)

Every block header in FUZION ends with a colon :

This is a universal, consistent rule with no exceptions.

Why?

  • Visual consistency - You always know when a block is starting
  • No ambiguity - Parser knows "next line should be indented"
  • Reads naturally - Like saying "if this condition: then do this"
  • No confusion - Unlike Python which has exceptions, FUZION is strict

üîç What Counts as a "Block Header"?

Any line that introduces an indented block of code:

1. Function Definitions

function_example.fz
add x, y:           # ‚Üê Block header (ends with :)
    return x + y    # ‚Üê Block body (indented)

greet name:         # ‚Üê Header
    print "Hello"   # ‚Üê Body
    print name      # ‚Üê Body

2. Conditional Statements (if/else)

conditionals.fz
if age >= 18:               # ‚Üê Header
    print "Adult"           # ‚Üê Body

if score > 90:              # ‚Üê Header
    print "Excellent"       # ‚Üê Body
else:                       # ‚Üê Header (else also needs :)
    print "Keep trying"     # ‚Üê Body

3. Match Statements

match.fz
match status:           # ‚Üê Header (match keyword)
    case "ok":          # ‚Üê Header (each case)
        print "Good"    # ‚Üê Body
    case "error":       # ‚Üê Header
        print "Bad"     # ‚Üê Body
    else:               # ‚Üê Header
        print "Unknown" # ‚Üê Body

4. Loops

loops.fz
# While loop
while count < 5:        # ‚Üê Header
    print count         # ‚Üê Body
    count = count + 1   # ‚Üê Body

# For loop
for i in 1 to 5:        # ‚Üê Header
    print i             # ‚Üê Body

# Infinite loop
loop:                   # ‚Üê Header
    get_input()         # ‚Üê Body

5. Type Definitions

types.fz
Person:                 # ‚Üê Header (type definition)
    name: text          # ‚Üê Body (fields)
    age: number         # ‚Üê Body

    greet:              # ‚Üê Header (method definition)
        print "Hi"      # ‚Üê Body

üé® Visual Pattern Recognition

The Pattern 
KEYWORD PARAMETERS:     # ‚Üê Always ends with :
    indented code       # ‚Üê Block starts here
    more code           # ‚Üê Still in the block
next line               # ‚Üê Back to outer level (no indent)

Real Examples

# Pattern 1: Function
calculate x, y:         # keyword=none, parameters="x, y", colon=:
    result = x + y      # indented = belongs to function
    return result       # still indented = still in function
print "done"            # not indented = outside function

# Pattern 2: If statement
if x > 0:               # keyword="if", condition="x > 0", colon=:
    print "positive"    # indented = belongs to if
print "continuing"      # not indented = outside if

‚ùå Common Mistakes

Mistake 1: Forgetting the Colon

‚ùå WRONG - No colon

    print "large"

‚úÖ CORRECT

    print "large"

üîÑ Nested Blocks (Colons Stack)

Each level of nesting needs its own colon:

# Function header: colon
calculate x, y:
    # If header: colon
    if x > 0:
        # Another if header: colon
        if y > 0:
            return x + y
        # Else header: colon
        else:
            return x - y
    # Else header: colon
    else:
        return 0

üéØ Why Not Use Braces {}?

Many languages use braces to define blocks. FUZION chose colons + indentation because:

  • Less visual noise - No closing braces
  • Forced readability - Indentation must be correct
  • Faster to type - One character vs two
  • Cleaner look - More like natural writing

JavaScript (braces)

function add(x, y) {
    if (x > 0) {
        return x + y;
    } else {
        return 0;
    }
}

FUZION (colons + indent)

add x, y:
    if x > 0:
        return x + y
    else:
        return 0

üéØ Key Takeaways

  • Colon = Block Incoming - Always signals indented code follows
  • Universal Rule - No exceptions, every block header has one
  • One Per Level - Each nesting level gets its own colon
  • Visual Consistency - Makes code predictable and scannable
  • Parser Signal - Tells FUZION "expect indent on next line"

üí° Quick Mental Check

When writing FUZION, ask yourself:

"Am I about to write indented code under this line?"

If YES ‚Üí Put a colon at the end
If NO ‚Üí No colon needed
